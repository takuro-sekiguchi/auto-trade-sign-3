## 1. プロジェクト概要

### 1.1 背景・戦略的意図

**市場の課題**

暗号資産への関心が高まる一方、多くの個人投資家は「いつ買い、いつ売るべきか」の判断に苦慮している。また、詐欺的な情報商材やシグナル配信サービスによる被害も後を絶たず、信頼できる情報源へのニーズが高まっている。

**法規制環境と市場機会**

| 項目 | 現状 |
|------|------|
| 現行法 | 暗号資産の現物取引に関する助言は、金融商品取引法上の「投資助言業」に該当しない |
| 将来見通し | 金融庁の方針により、将来的には規制強化（金商法適用）が見込まれる |
| 戦略的判断 | 現行法下で合法的にサービスを提供できる期間を「先行者利益を獲得できる機会」と位置づける |

**事業戦略**

1. 法規制が強化される前に、市場シェアと実績（トラックレコード）を確立する
2. ユーザー基盤と信頼性を構築し、将来的な正規ライセンス取得への足がかりとする
3. 「情報提供」の範囲内で最大限の価値を提供し、投資助言業に該当しない設計を徹底する

---

### 1.2 目的・ゴール

**ミッション**

> 「投資の民主化」と「安全な資産形成」の支援

高度な市場分析を、専門知識がなくても活用できる形で提供し、すべての人が資産形成の機会を得られる世界を目指す。

**プロダクトゴール**

AIによる高度な市場分析を、初心者でも直感的に理解・実行できる「推奨保有比率（ポートフォリオ）」として提供し、ユーザーの投資判断をエンパワーメントする。

**ビジネスゴール**

| ゴール | 詳細 |
|--------|------|
| ユーザー獲得 | フリーミアムモデルによる早期のユーザー基盤構築 |
| 収益基盤 | サブスクリプション型（SaaS）による持続可能な収益構造の確立 |
| ブランド確立 | 信頼性の高い情報提供者としてのポジション獲得 |

---

### 1.3 スコープ

**✅ 対象範囲（In-Scope）**

| カテゴリ | 内容 |
|----------|------|
| 対象資産 | ビットコイン（BTC/JPY）の現物取引 |
| コア機能 | AIによる「推奨保有比率」の算出・配信（例: BTC 80% / JPY 20%） |
| プラットフォーム | PWA（Progressive Web App）を主軸としたWebアプリケーション、レスポンシブ対応（PC・スマホ両対応） |
| 補助機能 | 過去のパフォーマンスシミュレーション、トレンド判定（上昇/下落/横ばい）、参考価格（決済/撤退ライン）の提示 |

**❌ 対象外（Out-of-Scope）**

| カテゴリ | 内容 | 除外理由 |
|----------|------|----------|
| 自動売買（Bot） | APIによる取引所連携・自動執行 | 法的リスク、技術的複雑性 |
| デリバティブ取引 | 証拠金取引・レバレッジ取引 | 投資助言業への該当リスク |
| アルトコイン | ETH等の他暗号資産 | Phase 2以降で検討 |
| ネイティブアプリ | iOS/Androidアプリ | App Store審査リスク回避 |

## 2. ターゲットユーザー

### 2.1 ペルソナ

**メインペルソナ: 「かつて損をした」再チャレンジ層**

過去にビットコイン投資で失敗した経験を持ち、次こそは成功したいと考えているユーザー。

| 項目 | 詳細 |
|------|------|
| 属性 | 20代後半〜30代、会社員、年収400〜600万円 |
| 投資経験 | 過去にビットコインを購入したが、高値掴みや狼狽売りで損失を経験し撤退 |
| 課題 | 「また上がりそうだけど、いつ入ればいいか怖い」「仕事中はチャートを見れないから急変に対応できない」 |
| ニーズ | ずっとチャートに張り付かなくても、プロ並みの判断で「売り時・買い時」を教えてくれるパートナーが欲しい |
| 心理 | 失敗のトラウマがあるため、信頼できる根拠がないと動けない |

**サブペルソナ: 堅実な資産形成層（NISA/iDeCo併用）**

すでに資産運用を行っているが、暗号資産はリスクが高すぎて手を出せずにいるユーザー。

| 項目 | 詳細 |
|------|------|
| 属性 | 30代〜40代、既婚、資産運用に関心が高い |
| 投資経験 | つみたてNISA、iDeCo等を運用中。暗号資産は未経験または少額保有 |
| 課題 | ポートフォリオの一部（5〜10%）に暗号資産を入れたいが、ボラティリティが高すぎて管理しきれる自信がない |
| ニーズ | 「全力投資」ではなく、適切な比率管理でリスクを抑えながら運用したい |
| 心理 | 堅実志向。感覚ではなくロジカルな根拠に基づいて判断したい |

---

### 2.2 ユースケース

**シーン1: 通知からのポートフォリオ調整（週1〜2回）**

| 項目 | 内容 |
|------|------|
| トリガー | LINE/Discordに「AI推奨比率が変更されました（BTC 50% → 80%）」という通知が届く |
| アクション | アプリを開き、現在の推奨比率と変更理由を確認。その後、自身の取引所アプリで成行注文を出す |
| ユーザー価値 | 思考停止で従うだけで、最適なポジション調整が完了する |

**シーン2: 放置運用のセットアップ（エントリー時）**

| 項目 | 内容 |
|------|------|
| トリガー | 新規エントリーや買い増しを行った直後 |
| アクション | アプリに表示された「決済目標価格」と「撤退ライン」を参考に、取引所アプリで指値（売り）と逆指値（ストップロス）を設定 |
| ユーザー価値 | 急変時も自動で利確・損切りされる安心感を得て、仕事や睡眠に集中できる |

**シーン3: 信頼性の確認と有料プランへの移行（検討フェーズ）**

| 項目 | 内容 |
|------|------|
| トリガー | 無料版でサイン通りにトレードし、利益が出た（または損失を防げた）とき |
| アクション | 「なぜこの判断なのか」を知りたくなり、過去のシミュレーション結果やAI判断理由を見るために有料プランに登録 |
| ユーザー価値 | サービスへの信頼が高まり、より深い情報を得ることで自信を持って運用できる |

## 3. 機能要件

### 3.1 コンセプト

> **「結論（比率）は無料で、根拠（AI脳）は有料で」**

ユーザーに行動（売買）させるための「結論」は惜しみなく提供し、その精度とロジックに価値を感じたユーザーから対価をいただく設計とする。

---

### 3.2 主要機能一覧

| ID | 機能名 | 詳細 | 優先度 |
|----|--------|------|:------:|
| F-001 | **推奨ポートフォリオ表示** | 現在の最適な「BTC:日本円」の比率を表示。例: 「強気相場です。BTC比率を80%まで引き上げてください」 | Core |
| F-002 | **参考価格・オーダーガイド** | エントリー後の「決済目標（利確）」と「撤退ライン（損切）」の価格を表示。「指値を入れて放置」というUXを実現 | Core |
| F-003 | **AIマーケット解説** | なぜその比率なのか？（テクニカル分析、市場動向など）の要約を表示。**冒頭のみ無料、全文は有料** | Core |
| F-004 | **パフォーマンス・シミュレーション** | 「過去90日、AIに従っていたら資産はどう増えたか」のグラフ。サービスの信頼性を担保する証拠 | 高 |
| F-005 | **シグナル履歴チャート** | ローソク足チャート上に過去の「買い/売り」ポイントをマーカー表示。時間足切り替え対応（1時間/4時間/1日/1週） | 高 |
| F-006 | **通知連携** | 比率変更や急変時にプッシュ通知。**リアルタイム通知は有料プラン限定** | 高 |
| F-007 | **免責・リスク確認** | 初回起動時に免責事項への同意モーダルを表示。法的リスク回避のため必須 | 必須 |
| F-008 | **アカウント登録・認証** | 有料プラン購入時のみ必要。メール/パスワード、SNS認証（Google, Apple）対応 | 必須 |
| F-009 | **決済機能** | サブスクリプション決済（月額/年額）。7日間返金保証対応 | 必須 |

---

### 3.3 画面構成

3画面 + 決済フローのシンプルな構成とする。

**メイン画面（3タブ）**

| 画面名 | 主な機能 |
|--------|----------|
| ホーム | 推奨ポートフォリオ、参考価格、ローソク足チャート、資産推移、AI解説（冒頭のみ） |
| 実績 | パフォーマンス・シミュレーション、調整履歴（理由コメント付き） |
| 設定 | アカウント情報、通知設定、プラン情報、ヘルプ・サポート |

**補助画面**

| 画面名 | 用途 |
|--------|------|
| 料金プラン | プラン比較、購入ボタン |
| アカウント登録 | 有料化時のみ表示。メール/パスワード、SNS登録 |
| 登録完了 | 購入完了確認、アンロック機能の案内 |
| 免責同意 | 初回起動時のモーダル |

## 4. ビジネスモデル・課金体系

### 4.1 フリーミアムモデル

**基本方針:** 
- 無料ユーザーは登録不要で利用可能
- 有料プラン購入時のみアカウント登録を要求
- 無料でもサービスの価値を体験できる設計

### 4.2 プラン比較

| 機能 | Free | Standard | Premium |
|------|:----:|:--------:|:-------:|
| 最新サイン閲覧 | ✓ | ✓ | ✓ |
| ローソク足チャート | ✓ | ✓ | ✓ |
| 過去30日の履歴 | ✓ | ✓ | ✓ |
| 資産シミュレーション | ✓ | ✓ | ✓ |
| 全期間の履歴 | - | ✓ | ✓ |
| サイン更新通知 | - | ✓ | ✓ |
| AI判断理由 | - | - | ✓ |
| 優先サポート | - | - | ✓ |

### 4.3 価格設定

| プラン | 月額 | 年額（2ヶ月分お得） |
|--------|------|-------------------|
| Free | ¥0 | ¥0 |
| Standard | ¥2,980 | ¥29,800 |
| Premium | ¥9,800 | ¥98,000 |

### 4.4 返金ポリシー

- 7日間返金保証
- 返金条件：初回購入後7日以内に申請

## 6. 法的・コンプライアンス要件

### 6.1 投資助言業との関係

**重要:** 本サービスは投資助言業に該当しないよう、以下の設計方針を採用する。

| 項目 | 対応方針 |
|------|----------|
| 表示内容 | 具体的な売買金額ではなく「比率」で提示 |
| 用語 | 「推奨」「参考」「シミュレーション」を使用 |
| 免責 | 全画面に免責事項を表示 |
| 判断主体 | ユーザー自身の判断と責任を明示 |

### 6.2 免責事項

以下の文言を全画面に表示：
> ※ 本サービスは情報提供を目的としたものであり、投資助言ではありません。投資判断はご自身の責任で行ってください。

### 6.3 個人情報保護

| 項目 | 対応 |
|------|------|
| プライバシーポリシー | 必須、登録時に同意取得 |
| 個人情報の利用目的 | サービス提供、決済処理、連絡のみ |
| 第三者提供 | 原則禁止（決済代行を除く） |
| 退会時 | 個人情報を削除 |

### 6.4 利用規約

- サービスの利用条件を明記
- 禁止事項、免責事項を含む
- 弁護士レビューを推奨

## 8. リスクと対策

| リスク | 影響度 | 対策 |
|--------|:------:|------|
| 法的リスク（投資助言業該当） | 高 | 弁護士レビュー、免責事項の徹底 |
